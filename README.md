# 夜巡录：荒庙篇

《夜巡录：荒庙篇》是一个志怪题材的卡牌构筑 roguelike 第一关竖切 demo。玩家扮演游方夜巡人，带着半枚城隍印进入永宁县外的荒庙夜路，在普通战、精英战、怪事、阴市、休整与 Boss 战之间选择路线，通过符箓、剑诀、香火、法门和遗物构筑一套能活到天明的牌组。

当前版本已经能完整跑通第一关：从标题页选择难度，进入路线地图，经历战斗与事件，获得卡牌、金币和遗物，最终挑战荒庙正殿 Boss。

## 在线试玩

公开试玩地址：

```text
https://card-game-night-patrol.pages.dev/
```

浏览器打开即可游玩，不需要安装客户端。建议使用桌面端 Chrome、Edge 或 Safari 访问，以获得更稳定的拖拽出牌体验和完整视听表现。

## 项目定位

这个项目不是一个静态概念稿，而是一个可玩的垂直切片。它的目标是验证三个核心问题：

- 志怪题材是否能和卡牌构筑 roguelike 的路线、事件、遗物、战斗循环自然融合。
- 以 React 承载复杂 UI、以 Phaser 承载战斗舞台，是否能兼顾开发效率和游戏表现。
- 在有限内容量下，是否能形成可重复游玩的构筑张力，而不是只靠单次剧情体验。

从产品角度看，它适合作为后续扩展的 playable prototype：可以继续增加章节、敌人、卡池、遗物、事件、动画演出、数值平衡和桌面发行流程。

## 核心体验

### 路线选择

玩家每一层从若干可选节点中选择前进方向。不同路线会影响资源节奏：

- 普通战：获得金币和卡牌奖励，是构筑成长的主要来源。
- 精英战：战斗风险更高，胜利后更容易获得强力回报。
- 怪事：提供事件选择，可能改变生命、牌组或资源。
- 阴市：购买卡牌、遗物，或花钱移除牌。
- 休整：回复生命，或升级一张牌。
- Boss：第一关最终挑战。

### 战斗系统

战斗采用回合制卡牌机制。每回合玩家获得能量并抽牌，可以用攻击牌造成伤害，用技能牌获得格挡、抽牌或施加状态，用法门牌建立长期能力。

核心资源与状态：

- 符印：施加在敌人身上的标记，可以持续配合，也可以被特定卡牌引爆。
- 香火：战斗内临时资源，用来支撑请神、雷法和爆发牌。
- 格挡：抵消当回合受到的攻击伤害。
- 法门：打出后在本场战斗持续生效。
- 虚弱与易伤：改变攻防效率，制造节奏窗口。
- 遗物：改变开局资源、抽牌、伤害、防御或牌组循环方式。

构筑目标不是拿单张最强牌，而是让牌组循环起来：抽得更多、费用更顺、弃牌能回卷、符印能叠高并爆发。

### 操作方式

- 鼠标拖拽攻击牌到敌人区域出牌。
- 鼠标拖拽技能牌和法门牌到玩家区域出牌。
- 点击“结束回合”进入敌人回合。
- 地图界面点击可选节点前进。
- 顶部按钮可以回到首页、静音、查看关于信息或重新开始。

## 当前内容

当前版本包含：

- 1 名主角：游方夜巡人。
- 20 张左右可用卡牌。
- 多种敌人：灯笼精、披蓑水鬼、庙祝尸、山魈、黑坛术士、画皮影、山君。
- 多种节点：普通战、精英战、Boss 战、怪事、阴市商店、休整。
- 多套奖励：卡牌奖励、金币奖励、遗物奖励。
- 多种构筑机制：符印、香火、格挡、虚弱、易伤、法门、消耗。
- 三档难度：演示、标准、劫难。
- React UI、Phaser 战斗舞台、背景音乐、攻击音效、技能音效、胜利结算视频和静态 fallback。

## 设计原则

### 题材表达

游戏选择“荒庙夜巡”作为第一关主题，避免一开始铺开过大的世界观。荒庙、城隍印、符箓、香火、阴市、山君等元素都服务于可玩系统，而不只是氛围文本。

### 构筑节奏

第一关目标是让玩家在短时间内理解三个问题：

- 我现在的牌组靠什么赢。
- 我下一步路线应该补资源、补牌、删牌还是升级。
- 我是否愿意为更高收益承担更高战斗风险。

### 交互策略

战斗采用拖拽出牌，让卡牌行为更接近实体卡牌操作。地图、商店、休整、奖励界面保持清晰按钮式操作，减少非战斗环节的学习成本。

### 可扩展性

游戏数据主要集中在内容定义和引擎逻辑中，UI、战斗状态、卡牌效果、敌人意图和奖励流程分层组织，方便后续继续扩充内容。

## 技术架构

### 技术栈

- React 18：复杂 UI、卡牌、地图、事件、商店、奖励和结算界面。
- Phaser 3：战斗舞台、背景、角色、敌人、雾气粒子、受击反馈和镜头震动。
- TypeScript：游戏状态、卡牌、敌人、遗物、事件和路线类型。
- Vite：本地开发服务器和前端构建。
- Electron：桌面客户端打包能力。
- Cloudflare Pages：公开网页试玩部署。
- GitHub Actions：桌面客户端构建流程。

```text
  styles.css           # 全局视觉、布局和响应式样式

    audio.ts           # 音效和音乐控制

  FIRST_ACT_DEMO_ROADMAP.md

  PACKAGING_DISTRIBUTION.md
```

```bash

```

```bash
npm run dev
```

默认访问地址：

```text
http://127.0.0.1:5173
```



```bash
npm run preview
```

```bash

```



macOS 版本在 demo 阶段可能是 ad-hoc 签名包，没有 Apple 公证。如果系统提示“无法验证开发者”，需要右键点击应用，选择“打开”，再确认一次。

## Cloudflare Pages 部署

当前项目已经适合静态部署。推荐部署方式：

1. 本地执行：

   ```bash
   npm run build
   ```

2. 将 `dist/` 上传到 Cloudflare Pages。

3. 或使用 Wrangler：

   ```bash
   npx wrangler pages deploy dist --project-name card-game-night-patrol --branch main
   ```

线上正式地址：

```text
https://card-game-night-patrol.pages.dev/
```

当前构建包含完整图片、音频、视频资源。部署前建议检查：

```bash
du -sh dist
find dist -type f -size +24M -print
```

如果没有超过 Cloudflare Pages 免费上传单文件限制的大文件，就可以直接部署。

## 测试与验收建议

正式发布前建议至少检查以下路径：

- 首页能正常加载，难度选择可切换。
- 点击“开始夜巡”能进入地图。

- 普通战能进入战斗，手牌能拖拽出牌。

- 升级/删牌/商店等长列表页面能滚动到底部。
- 音频按钮能静音/恢复。
- 胜利视频和 poster fallback 能正常显示。
- Cloudflare Pages 线上链接与本地 `dist/` 内容一致。

## 后续路线

建议按优先级推进：

1. 数值平衡：压测三档难度，减少无效卡和必拿卡。
2. 内容扩展：增加第二关主题、敌人池、事件池和 Boss。
3. 新角色：加入不同初始牌组、专属机制和成长路线。
4. 交互打磨：补充拖拽提示、目标高亮、状态解释和结算节奏。
5. 性能优化：压缩视频体积，拆分大 chunk，优化首屏加载。
6. 发行准备：替换或复核全部素材授权，完善图标、签名、公证和安装包体验。
